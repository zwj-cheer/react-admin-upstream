# 上游工作流

## 创建下游项目

1. 从稳定的模板标签创建新仓库。
2. 将项目仓库设为 `origin`。
3. 将模板仓库添加为 `upstream`。
4. 只修改文档中列出的项目侧文件与运行时部署配置。
5. 在第一个项目提交前运行全部质量门禁。

## 升级

拉取 upstream 标签并合并选定的稳定标签，而不是持续变动的 main 分支。先阅读匹配的升级说明，
解决冲突时 `src/project/` 内以项目侧文件优先。

每个稳定发布都记录：

- 破坏性变更；
- 下游必须执行的操作；
- 已知冲突热点；
- 验证命令与视觉基线影响。

在发布新的稳定标签之前，独立的示例下游项目会先从上一个稳定标签完成升级。
